# Publication Code Notes

This repository is intended to support transparent reproduction of the public-data analyses reported in the manuscript.

## Recommended GitHub Structure

- Keep the branch-specific processing pipelines in `pipelines/`.
- Keep final model fitting in `scripts/04_meta_analysis/`.
- Keep cross-omic convergence and enrichment scripts in `scripts/05_convergence_pathway/`.
- Keep figure scripts in `scripts/06_figures/`.
- Keep setup, run-order, package, and scope documentation in `docs/`, `config/`, and `manifests/`.

## Final Meta-Analysis Scripts

The final random-effects meta-analysis step is deliberately split into small scripts:

- `scripts/04_meta_analysis/lib/metafor_helpers.R` contains shared functions for reading dataset-level Hedges' g tables, fitting DerSimonian-Laird models with `metafor`, calculating FDR, and classifying modified Knapp-Hartung interval support.
- `scripts/04_meta_analysis/01_fit_methylation_models_metafor.R` fits all DNA methylation primary, sensitivity, descriptive, and exploratory models.
- `scripts/04_meta_analysis/02_fit_expression_models_metafor.R` fits all gene-expression primary and sensitivity models.
- `scripts/04_meta_analysis/00_run_all_meta_analysis_models.R` is only a convenience wrapper that runs the methylation and expression scripts in sequence.

This separation is preferable to a single all-in-one meta-analysis script because it makes the tissue and omic-layer boundaries clearer to readers.

## Source Inputs

Large public repository files are not duplicated in the code repository. Scripts either download public sources where implemented or allow users to provide a local source-cache folder through documented environment variables.

Controlled-access datasets are not analysed by these scripts.

## Important Interpretation Boundaries

- Single-dataset outputs are descriptive and are not pooled meta-analyses.
- GSE36315 is treated as a custom-annotated sensitivity analysis only.
- Cross-omic convergence is summary-level and tissue-matched; it is not participant-level regulatory coupling.
- Where public repositories provide processed matrices rather than raw intensity files, the workflow starts from those processed matrices.

## Repository Scope Check

The code comments, folder names, and run-order documentation use publication-facing language. The final reported pooled models are generated by the scripts in `scripts/04_meta_analysis/`.
